# Philosophy

SolarUI est construit sur quelques principes intentionnels. Pas des conventions arbitraires — chacun répond à un vrai problème visuel ou d'interaction. Cette page les documente pour que le système reste cohérent au fil du temps.

---

## Built on shadcn/ui — open code, no lock-in

SolarUI repose sur shadcn/ui, et ce choix est philosophique autant que technique. shadcn/ui n'est pas une librairie de composants au sens classique du terme : c'est une façon de construire ta propre librairie. Plutôt que d'installer un package opaque et de te battre contre ses API, tu reçois le code source directement dans ton projet. Tu le possèdes. Tu peux le lire, le modifier, l'adapter — sans workarounds, sans wrapper, sans lock-in.

> "This is not a component library. It is how you build your component library."

C'est exactement ce que SolarUI fait : il s'appuie sur cette fondation ouverte pour construire un système de design cohérent, que tu contrôles entièrement.

---

## Controls share a single height

Buttons and inputs must always have the same height. When a button sits next to an input, they form a single visual unit. If their heights differ, that unit breaks — the eye reads two separate elements instead of one composed control.

<DoAndDont>
  <Do label="Do — same height">
    <div className="flex w-full max-w-xs gap-2">
      <Input placeholder="Email address" />
      <Button>Subscribe</Button>
    </div>
  </Do>
  <Dont label="Don't — mismatched heights">
    <div className="flex w-full max-w-xs items-center gap-2">
      <input
        placeholder="Email address"
        className="flex h-10 w-full rounded-md border border-(--color-input) bg-(--color-background) px-3 text-sm"
      />
      <button className="inline-flex h-6 shrink-0 items-center rounded-lg bg-default-12 px-2.5 text-xs font-medium text-brand-1">
        Subscribe
      </button>
    </div>
  </Dont>
</DoAndDont>

This is enforced at the token level. Both `Button` (default size) and `Input` use the same `h-control` class, which maps to the `--height-control` CSS variable. Changing one automatically changes both.

```css
/* globals.css */
--height-control: 2.25rem; /* 36px */
```

```tsx
// button.tsx  →  size default
"h-control gap-1.5 px-2.5 ..."

// input.tsx
"flex h-control w-full ..."
```

This also applies across sizes: a small button (`size="sm"`) paired with a small input should share the same height token for that size tier.
